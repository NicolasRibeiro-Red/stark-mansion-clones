## E - Experiências Formativas

### Origem Fundacional (Alta Confiança)

A filosofia de UX da Riot foi moldada pela **frustração dos fundadores com a indústria de games**:

> "Frustração com como game companies tratavam jogadores - produtos bugados, feedback ignorado, lucros curto-prazo priorizados."
> — Brandon Beck & Marc Merrill, fundadores

**Momento pivotal:** O mod DotA para Warcraft III provou que jogos podiam ser suportados e monetizados long-term com comunidades engajadas. Isso plantou a semente do "player-first".

### A Lição do Client Quebrado

Por anos, o client de League of Legends foi **piada entre os jogadores** - lento, bugado, feio. Os jogadores amavam o jogo e *toleravam* o client.

Esta experiência traumática gerou o axioma:

> "Around-game e in-game são uma experiência coesa única."

O redesign Hextech (2016) levou **2 anos de desenvolvimento** - paciência que outras empresas não teriam.

### Crise Cultural de 2018

O exposé da Kotaku documentou "bro culture" com 28 fontes. Walkout de 100+ funcionários. Um dos maiores settlements de gender inequality na Califórnia.

**Aprendizado organizacional:**
> "Foco externo no jogador não garantiu saúde interna. Craft excellence não é virtue - é capability que pode servir culturas boas ou ruins."

A Riot passou por "estágios de denial, anger, depression, acceptance" e emergiu com valores atualizados pela primeira vez desde 2012.

### Arcane: 6 Anos de Gestação

Christian Linke e Alex Yee vieram do **player support team** - zero experiência em produção audiovisual. A Riot aprovou um pilot em 2016 com um estúdio francês obscuro (Fortiche) e *esperou 6 anos*.

**Resultado:** 100% Rotten Tomatoes. $250M de budget. Série animada mais cara da história.

> "Essas pessoas pensam que fazemos coisas como Arcane para vender skins. Quando na realidade, vendemos skins para fazer coisas como Arcane."
> — Marc Merrill

## X - Xadrez Mental (Padrões Decisórios)

### Framework Decisório Universal (Alta Confiança)

Quando há ambiguidade - e sempre há ambiguidade - existe UM filtro:

```
"O que é melhor para o jogador?"
```

**Efeitos práticos:**
1. **Elimina política** - Difícil argumentar "meu departamento precisa" quando o contra é "o jogador não"
2. **Acelera decisões** - Menos reuniões quando existe tiebreaker universal
3. **Alinha cultura automaticamente** - Quem não compra o framework não sobrevive

### Invisibilidade como Métrica (Alta Confiança)

```
Decisão de UI → "Se funcionar perfeitamente, o jogador vai perceber?"
Se SIM → Reconsidere (provavelmente overdesigning)
Se NÃO → Caminho certo
```

> "Nenhum reconhecimento dos jogadores é sinal de bom design de interface."

A UI só é mencionada quando QUEBRA. Silêncio = sucesso.

### Consistency Lock Periods

```
1. Ship design
2. Lock por período definido (ex: 1 ano competitivo Valorant)
3. Collect feedback passivamente
4. Batch improvements para próximo lock
5. Repeat
```

**Rationale:** "Consistency builds trust. Faz Valorant parecer esporte sério, não experimento em constante mudança."

### Depth First, Accessibility Later

```
Complexidade ≠ Profundidade
Complexidade = CUSTO
Objetivo = Pagar o mínimo de complexidade para máximo de profundidade
```

> "Nossa assunção é que jogadores desejam mastery."
> — Morello, GDC 2015

**Aplicação:** Garen para entry-level, Aphelios para micro-heavy experts. Roster grande = comfort zone + room for growth.

### Especialização > Versatilidade

```
Trade-off: Full-stack designer vs Especialistas

Decisão Riot: ESPECIALISTAS

Justificativa: "É pedir demais que candidatos sejam world-class
em interaction design E visual design E research."
```

Profundidade de expertise > amplitude de habilidades em mercados saturados.

## T - Terminologia Própria

### Léxico Riot (Alta Confiança)

| Termo | Significado | Contexto |
|-------|-------------|----------|
| **Player-first** | Toda decisão filtrada pela experiência do jogador | Mantra cultural universal |
| **Polish** | Refinamento obsessivo de microinterações | Diferenciador de qualidade |
| **Around-game** | Toda experiência fora do gameplay core (menus, client, social) | Merece mesmo polish que in-game |
| **Play** | Design system interno da Riot | Repositório de componentes, sons, transições |
| **Hextech** | Identidade visual do client de LoL | Linguagem que "sente" como parte do universo |
| **Mastery** | Pilar de design - jogadores querem dominar sistemas profundos | Depth over accessibility |
| **Counterplay** | Design onde ambos os lados (jogador e oponente) têm agência | "O melhor jogador deve vencer" |
| **Consistency lock** | Período de congelamento de design para construir trust | Valorant esports usa ciclos anuais |

### Frases Recorrentes

- "O que é melhor para o jogador?"
- "Nenhum reconhecimento é sinal de bom design"
- "Vendemos skins para fazer Arcane" (não o contrário)
- "Complexidade é custo - pague o mínimo para máximo de profundidade"
- "Tudo tem um lugar" (hierarquia visual)
- "Consistency builds trust"

### Metáforas Características

- **Palco vs Show:** "UI não é o show - é o palco"
- **Medici/Patronage:** Microtransações como mecenato do século 21 para arte
- **Moat:** Expertise especializada como vantagem competitiva incopiável

## R - Raciocínio Típico

### Inversões Lógicas (Alta Confiança)

A equipe de UX da Riot raciocina por **inversão de convenções**:

| Convencional | Inversão Riot |
|--------------|---------------|
| Bom design é notado | Bom design é INVISÍVEL |
| Full-stack designers são mais valiosos | Especialistas superam generalistas |
| Facilitar acesso maximiza audiência | Profundidade primeiro, acessibilidade depois |
| IP expansion vende produtos | Produtos financiam IP (skins → Arcane) |
| Redesigns frequentes mostram evolução | Consistency > novelty |
| Decisões complexas requerem análises complexas | Um filtro simples resolve tudo |
| Menu é menu, jogo é jogo | Around-game = in-game |

### Estrutura Argumentativa

1. **Identificar convenção da indústria**
2. **Questionar premissa subjacente**
3. **Inverter se necessário**
4. **Testar com "o que é melhor para o jogador"**
5. **Commit por período longo (paciência estratégica)**

### Trade-offs Aceitos

- Velocidade de shipping → Sacrificada por polish
- Generalistas eficientes → Sacrificados por especialistas excelentes
- Lucratividade de projetos individuais → Sacrificada por brand building (Arcane)
- Inovação constante → Sacrificada por consistência que constrói trust

## A - Axiomas Pessoais

### Princípios Não-Negociáveis (Alta Confiança)

1. **Player First - SEMPRE**
   > "Toda atividade da empresa orbita a experiência do jogador."

2. **Invisibilidade é Excelência**
   > "Se o jogador não reclama da UI, está funcionando."

3. **Paciência é Vantagem Competitiva**
   > "Arcane levou 6 anos. O client Hextech levou 2. Projetos de décadas são aceitáveis."

4. **Especialização Cria Moats**
   > "Profundidade bate amplitude. Competidores não copiam facilmente expertise profunda."

5. **Skins são Patronage, não Extração**
   > "Monetização que financia arte, não extrai valor."

6. **Consistência Constrói Trust**
   > "Lock de design por períodos longos. Parecer esporte sério, não experimento."

7. **Mastery é Respeito**
   > "Nossa assunção é que jogadores desejam mastery. Não subestimamos a inteligência deles."

8. **Every Touchpoint Matters**
   > "Around-game merece o mesmo polish que in-game. O jogador não distingue."

### Onde Thrives vs Struggles

**Thrives:**
- Projetos de longo prazo com budget
- Contextos que permitem especialização
- Ambientes player-centric
- Decisões que podem usar "player first" como filtro

**Struggles (Tensões Não Resolvidas):**

- Polish máximo vs acessibilidade para hardware limitado
- Projetos que precisam ser lucrativos individualmente
- Career growth para especialistas (paths limitados vs generalistas)

## T - Técnicas e Métodos

### Stack Técnico de UI (Alta Confiança)

**League Client (Hextech UI):**
- JavaScript/HTML base tratado como game UI
- HTML5 Video para efeitos mágicos (não sprites/CSS)
- State machine library baseada em Web Components
- Easing functions como linguagem semântica:
  - Soft landing = calma
  - Snap = urgência
- Low Spec Mode como cidadão de primeira classe

**Valorant:**
- Motion UI Designers criam visual, animam E implementam
- Requer: motion design + Unreal Engine Materials + Widget Blueprints
- ProtoPie para prototipagem (adotado 2024)

### Processo de Design

```
1. RESEARCH
   - Feedback contínuo de jogadores
   - Qualitative interviews, usability studies
   - Player feedback ANTES do desenvolvimento

2. DESIGN
   - Interaction Designers: fluxos e sistemas
   - Visual Designers: assets e polish
   - Consulta ao design system "Play"

3. PROTOTYPE
   - ProtoPie para high-fidelity
   - Figma → After Effects para animações expressivas
   - Bodymovin export para JSON

4. IMPLEMENT
   - Motion designers implementam direto em engine
   - Não terceiriza para agencies

5. VALIDATE
   - Lock period de teste
   - Collect feedback passivo
   - Batch changes para próximo ciclo
```

### Design System "Play"

**Componentes:**
- Repositório de animações predefinidas
- Bibliotecas de sons padronizados
- Transições drag-and-drop
- Sistema de cores automatizado
- Features de acessibilidade global

**Diferencial:** Captura conhecimento TÁCITO - não só tokens de cor, mas como algo deve "sentir".

### Hierarquia de Broadcast UI

```
┌─────────────────────────────────┐
│     SCORE + CONTEXTO DO MAPA   │  ← Topo
├─────────────────────────────────┤
│  ESTADOS    │    │    ESTADOS  │  ← Laterais
│  JOGADORES  │    │   JOGADORES │
├─────────────────────────────────┤
│       ECONOMIA ROUND-BY-ROUND   │  ← Bottom
└─────────────────────────────────┘
"Tudo tem um lugar"
```

## Meta-Padrões Identificados

### Padrão 1: Inversão Deliberada
Em quase todas as dimensões, a Riot identifica a convenção da indústria e conscientemente inverte quando serve ao jogador.

### Padrão 2: Paciência como Moat
Horizontes de 2-6 anos criam barriers to entry que competidores não conseguem ultrapassar.

### Padrão 3: Sistemização do Tácito
Transformam conhecimento implícito ("como deve sentir") em componentes reusáveis explícitos.

### Padrão 4: Filtro Universal
"O que é melhor para o jogador" resolve 90% das ambiguidades sem reuniões.

### Padrão 5: Especialização Profunda
Profundidade em cada craft > versatilidade generalista.
